# 在线SDF生成端到端训练计划

## 目标
创建一个使用在线SDF生成的端到端训练脚本，直接从深度图实时计算TSDF，无需预先生成SDF真值。

## 背景
- 原`real_tartanair_dataset.py`已经实现了在线TSDF生成
- 使用多帧深度图融合生成局部TSDF
- 支持不同分辨率（0.04m, 0.08m, 0.16m）
- 使用稀疏张量表示以节省内存

## 任务分解

### 1. 创建在线SDF训练数据集
- 基于`real_tartanair_dataset.py`创建简化版本
- 支持TartanAir样本数据格式
- 实现实时TSDF计算
- 添加数据增强

### 2. 创建端到端训练脚本
- 加载StreamSDFFormer模型
- 使用在线生成的SDF作为监督
- 实现训练循环
- 添加验证和监控

### 3. 优化训练流程
- 内存优化（批次大小、裁剪尺寸）
- 训练稳定性（梯度裁剪、学习率调度）
- 性能监控（损失、梯度、内存）

### 4. 测试与验证
- 运行完整训练循环
- 验证模型收敛
- 检查内存使用
- 输出训练报告

## 技术细节

### 在线SDF生成流程
1. **数据加载**：加载RGB图像、深度图和位姿
2. **场景采样**：随机选择锚点，创建局部裁剪
3. **TSDF计算**：
   - 将采样点投影到每个相机视图
   - 计算有符号距离（深度差）
   - 多帧融合（取最小绝对距离）
   - 截断和归一化
4. **多分辨率处理**：生成0.04m, 0.08m, 0.16m分辨率
5. **稀疏表示**：转换为稀疏张量

### 训练策略
1. **损失函数**：SDF回归损失（L1或Huber）
2. **优化器**：AdamW，带权重衰减
3. **学习率**：余弦退火调度
4. **批次大小**：1（由于内存限制）
5. **训练时长**：3-5个epoch用于验证

### 模型配置
- **体素大小**：0.04m（基础分辨率）
- **裁剪尺寸**：(48, 48, 32) 体素
- **图像大小**：256x256
- **帧数**：5-10帧每序列

## 成功标准
- 训练损失在3个epoch内下降
- 梯度正常（无NaN或爆炸）
- GPU内存使用合理（<8GB）
- 模型参数正常更新
- 训练时间可接受（<30分钟）

## 风险与缓解
1. **内存不足**：
   - 减小裁剪尺寸
   - 减少帧数
   - 使用梯度累积

2. **训练不稳定**：
   - 梯度裁剪
   - 学习率预热
   - 权重初始化检查

3. **收敛缓慢**：
   - 调整损失函数
   - 优化采样策略
   - 增加数据增强

4. **数据质量问题**：
   - 验证深度图质量
   - 检查位姿准确性
   - 添加数据过滤
