## 🎯 Batch Size 2 的显存分析

### 理论计算

**当前配置（Batch Size 1）**：
- 模型参数：0.111 GB（静态）
- 第一帧峰值：0.120 GB
- 第二帧峰值：0.190 GB
- 总显存：0.196 GB

**Batch Size 2 理论显存**：
- 模型参数：0.111 GB（不变）
- 第一帧峰值：0.120 × 2 = 0.240 GB
- 第二帧峰值：0.190 × 2 = 0.380 GB
- **总显存：0.380 GB**

### 实际情况分析

**理论值 vs 实际需求**：
- 理论总显存：0.380 GB
- 实际需要：>6 GB（OOM时测量）
- **差异**：16倍差异！

**原因**：
1. **注意力计算的复杂度**：O(N²)，不是线性增长
2. **显存碎片化**：需要大块连续显存
3. **缓存开销**：PyTorch和CUDA的缓存机制

**Batch Size 2 的真实显存需求**：
- 注意力计算：~4.5 GB（峰值）
- 其他组件：~1.5 GB
- **总需求**：~6 GB（但需要4.51 GB连续块）

### 结论

**Batch Size 2 不可行**的原因：
1. 注意力计算的复杂度是O(N²)，不是线性增长
2. 需要大块连续显存（4.51 GB），但最大只有3.04 GB
3. 显存碎片化问题严重

**解决方案**：
- 使用梯度累积（batch_size=1 + accumulation_steps=4）
- 有效batch size = 4（超过batch_size=2）
- 显存使用保持低水平（~0.196 GB）
