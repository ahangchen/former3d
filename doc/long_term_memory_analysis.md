### 为什么之前OOM时显存占用5.26 GB？

**可能原因**：

1. **长期训练的累积效应**（30步 vs 几百步）
   - 我们的监控只有30步
   - 实际训练可能几百步
   - 可能存在长期累积

2. **真实训练的复杂度**
   - 我们的监控使用固定输入
   - 真实训练使用真实数据
   - 真实数据可能更复杂

3. **显存碎片化**
   - 长期训练导致显存碎片化
   - 需要更大的连续显存块
   - 虽然总显存够，但连续块不够

4. **梯度训练的差异**
   - 我们的前向传播是无梯度的
   - 真实训练需要梯度
   - 梯度会额外占用显存

## 🎯 最终答案

### 问题：为什么batch_size=2的显存比batch_size=1多那么多？

**答案**：

1. **短期监控（30步）**: 显示显存增长基本线性（1.79-3.90倍）
2. **长期训练**: 可能存在显存累积效应（需要更长时间监控）
3. **真实训练**: 比监控更复杂（梯度、真实数据、长时间）
4. **显存碎片化**: 长期训练导致需要更大的连续显存块

### 问题：即使体素数量增长，也远未达到9G的显存占用？

**答案**：

你说得对！基于30步的监控，batch_size=2的峰值显存只有0.462 GB，远未达到9 GB。

**但是**：
1. 真实训练更复杂（梯度、真实数据）
2. 长期训练可能有累积效应
3. 显存碎片化需要更大的连续块
4. 之前的OOM是在特定条件下发生的

**结论**：
- **短期（30步）**: 显存使用稳定，无泄漏，远未达到9 GB
- **长期（几百步）**: 可能存在累积效应（需要更长时间监控）
- **真实训练**: 可能比监控更复杂（需要实际训练验证）
